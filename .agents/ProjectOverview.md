# ゲームコンセプト
**『横スクロールハクスラ防衛ゲーム × リアルタイムCDカード × デッキ構築ローグライク』**

## 1. ゲームの概要
本作は、大軍勢がぶつかり合い砕け散る「横スクロール防衛ゲーム」の圧倒的なハクスラ感に、「ローグライク・デッキ構築」を融合させた新感覚タクティクスゲームです。
『Epic War 3』の気持ちよさと『Slay the Spire（StS）』の戦略性・適応力を併せ持ち、モバイル向けに最適化して構築します。

* **画面構成:** 画面下部40%を「手札・スペルスロット・マナ等のUI領域」、上部60%を横1レーン「戦闘画面」として使用。
* **世界観とグラフィック:** 中世ダークファンタジーのピクセルアート。大量のユニットが死ぬ際に派手に砕け散る（ゴア・破片表現）視覚的なカタルシスを追求。
* **ハクスラの視覚化と軽量化:** モバイル最適化のため、ユニット同士の「当たり判定（物理的な渋滞）」は排除。代わりに、重い一撃による「敵の吹き飛ばし」と、密集した敵が「連鎖的に砕け散る表現」で圧倒的な爽快感を生み出す。

## 2. コア・ゲームループ（リアルタイム戦闘）
本作の戦闘はリアルタイムで進行します。プレイヤーはカードのクールダウン（CD）管理と、スペルによる「時間停止」介入、そしてユニットの前後移動を駆使して戦場をコントロールします。敵拠点のHPをゼロにすれば勝利、自拠点が破壊されれば敗北です。

### リアルタイム戦闘の基本フロー
1. **手札とCD:** 戦闘開始時に手札が配られる。各カードには固有の「クールダウン（CD）」があり、CDが0になるとそのカードのユニットが自動的に召喚される。
2. **召喚とドロー:** ユニットが召喚されたカードは「捨て札」に送られ、即座に山札から1枚ドローされる。
3. **山札と捨て札の循環（StS仕様）:** 山札が0枚になった時点で、捨て札をシャッフルして新たな山札とする。
4. **敵ウェーブ:** 敵は時間経過に応じてウェーブ形式で出現。ステージが進むにつれ、敵の種類・数・タイミングが激化する。

### プレイヤーが出来ること（操作）
* **スペルの使用（時間停止）:** スペル発動時に「時間停止」が発生し、戦場全体がフリーズする。この間にスペル効果（範囲攻撃、バフ、デバフ等）の対象を指定して発動する。スペルはユニットカードとは別枠で管理され、山札の概念はなく、固定スロットに設定する方式。各スペルには個別のCDがある。
* **ユニットの前後移動:** 戦場にいる自軍ユニットをタップ/ドラッグで選択し、前進・後退の指示を出す。これにより陣形を能動的にコントロールできる。

## 3. カードシステム

### ユニットカード
* **CDベースの自動召喚:** 各カードには固有のCDがあり、CDが完了すると自動的にユニットが戦場に召喚される。プレイヤーが「いつ・どこに配置するか」を直接指定するのではなく、デッキの構築順や手札の巡りによって召喚タイミングが決まる。
* **手札の循環:** 召喚されたカードは捨て札 → 山札が尽きたら捨て札をシャッフルして山札に → 再びドロー、というStSと同じ循環を行う。
* **妨害カード（呪い・状態異常カード）:** 敵の攻撃やイベントにより、プレイヤーのデッキに妨害カードが混入する。これらは手札に来ると、CDが異常に長い、召喚されるユニットが敵に有利に働く、ドロー枠を無駄にする等、プレイヤーを不利にする。デッキの「汚染」として機能し、StSの「呪い」に相当する戦略的逆風要素。

### スペルカード（別枠管理）
* **スロット設定式:** スペルはデッキに混ぜず、戦闘前に固定スロット（2〜3枠程度）にセットする。
* **CD管理:** 各スペルには個別のCDがあり、CD完了後に任意のタイミングで発動可能。
* **時間停止との統合:** スペル発動時に時間が停止するため、状況を見極めてから効果を適用できる。これが本作における「思考の時間」を提供する。

## 4. 本作のキラーシステム（戦術を深める要素）
* **A. 「死」のリソース化:** 死亡時周囲ダメージ（自爆）や死亡時バフ散布により、ユニットの死を戦術に組み込むシナジーが可能。
* **B. 「寿命」と退場スキル:** 寿命（稼働時間）を迎えた瞬間に強力なスキルを放つ。寿命持ちには導火線UIが付き、「あと少し生き残ってくれ！」という祈りを視覚的に煽る。
* **C. 団子化対策とAoE（範囲攻撃）:** 当たり判定がないため、足の速い遠距離ユニットなどは一箇所に重なる（団子化）。敵側はこれに対する強烈なカウンターとして「密集地点を焼き払うAoE攻撃」を多用してくるため、ユニットの前後移動で散らす戦術が求められる。
* **D. カード同士のシナジー:** 特定のカードの組み合わせで相乗効果が発生する。例：「盾兵が前線を維持している間に弓兵のCDが完了し、安全に後衛火力を展開」「自爆ユニット → 死亡時バフ → 次の召喚ユニットが強化状態で登場」など、デッキ構築段階での"噛み合わせ"が戦果を大きく左右する。

## 5. デッキ構築と部隊編成システム（Out-of-Battle / StS仕様）
戦闘外の構造は基本的に『Slay the Spire』準拠とし、以下の要素で構成します。

### マップ移動
* StS式の分岐マップを採用。プレイヤーは各ノードを選択しながらボスまでの道のりを進める。
* ノードの種類：**通常戦闘 / エリート戦闘 / ショップ / 休憩 / イベント / ボス**

### カード獲得
* 戦闘勝利後、提示される複数枚のカードから1枚を選んでデッキに追加（スキップも可能）。

### ショップ
* カードの購入、不要カードの除去（有料）、スペルの購入、レリックの購入など。

### 休憩
* HPの回復、またはカードのアップグレード（強化）を選択。

### イベント
* ランダムイベント。リスクとリターンのある選択肢が提示される（例：HPを犠牲にレリック獲得、デッキに呪いカードが追加される代わりに強力なカードを入手、等）。

### レリック（アーティファクト）
* ボス討伐時やエリート戦闘、イベント等で獲得できるパッシブアイテム。
* ランダムにドロップするが、提示されたレリックに対して「今の自分のデッキをどう適応させるか」を考えるのが面白さの核心。
* 例：「全近接ユニットのノックバック率+15%」「CD完了時、そのユニットのHP+20%で召喚」「スペルのCD-20%だが最大HP-10」など。

## 6. ユニットおよび魔法のステータス定義
ユニットと魔法（設置型スペル等）は、以下の共通ステータスフォーマットで管理されます。

| ステータス名 | 英語表記 | 説明 |
| :--- | :--- | :--- |
| **CD（クールダウン）** | Cooldown | 手札に来てから召喚されるまでの待機時間（秒）。 |
| **体力** | HP / Health Points | 0になると砕け散る（死亡）。 |
| **攻撃力** | ATK / Attack Power | 敵に与える基本ダメージ量。 |
| **攻撃範囲** | AoE | ダメージを与える範囲。 |
| **防御力** | DEF / Defense | 敵から受けるダメージを軽減する割合、または固定値。 |
| **移動速度** | SPD / Movement Speed | レーンを進む速さ。 |
| **寿命** | Lifespan / Duration | 盤面に存在できる秒数。寿命が尽きるとHPに関わらず退場する。 |
| **スポーン時効果** | On-Spawn / Battlecry | 召喚された瞬間に発動する固有能力。 |
| **寿命時効果** | On-Expire | 寿命を迎えて退場する際に発動する能力。 |
| **死亡時効果** | On-Death / Deathrattle | HPが0になって敵に破壊された際に発動する能力。 |
| **ノックバック率** | Knockback Chance | 攻撃ヒット時、敵を吹き飛ばす確率（%）。 |
| **ノックバック力** | Knockback Power | 敵をどれだけ後方へ吹き飛ばすかの強さ。 |
| **ノックバック方向** | Knockback Direction | 吹き飛ばすベクトル（真後ろ、斜め上など）。 |
| **スタン率** | Stun Chance | 攻撃ヒット時、敵の行動を停止させる確率（%）。 |
| **スタン効果時間** | Stun Duration | スタン状態が持続する時間（秒）。 |
| **ノックバック耐性** | KB Resistance | 吹き飛ばしを無効化・軽減する割合（%）。 |
| **スタン耐性** | Stun Resistance | スタンを無効化・軽減する割合（%）。 |
| **特殊効果** | Special / Passive | 常時発動する固有能力。 |

## 7. 強敵・ボス戦の仕様
* **ボス戦:** 各マップの最終ノードにボスが待ち構える。ボスは通常の敵ウェーブとは比較にならない耐久力と攻撃力を持ち、固有のギミックで戦術を強制する。
* **戦術的スタンピード（HPトリガー）:**
    * **例:** ボスのHPが閾値（50%など）を下回った瞬間、大量の雑魚敵が急激に出現する。HPバーの「警告線（ドクロマーク）」により、プレイヤーは閾値を超えるタイミングを意図的にコントロールできる。

## 8. ターゲット層とプレイ体験の目標（UX目標）
* **ターゲット層:** 『Slay the Spire』のような思考のパズル的楽しさと、『Epic War 3』や無双系のような「画面を埋め尽くす敵をなぎ倒す」視覚的快感の両方を味わいたい、ミッドコア層以上のゲーマー向け。
* **至高の快感（脳汁ポイント）:**
    * 敵の大群が密集している箇所へ、時間停止スペルで絶妙なタイミングのAoEを叩き込み、敵陣がドミノ倒しのように連鎖的に砕け散る瞬間。
    * CDが絶妙に噛み合い、連続でユニットが召喚されて一気に戦線が押し返される"ラッシュ"の快感。
    * 「あと1秒で自拠点が落ちる！」という絶望的な状況で、デッキ構築のシナジー（死亡時効果の連鎖など）が決まり戦況が完全逆転するカタルシス。

## 9. アートスタイルとサウンドディレクション
* **ビジュアル:** ダークファンタジーテイストのピクセルアート。ユニットは小さめ（Epic War 3リスペクト）だが、攻撃エフェクトや死亡時のディスメンバーメント（四肢飛散、破片飛散）はあえて過剰に描き、爽快感を強調する。スマホ画面での視認性を高めるため、自軍・敵軍のシルエットやカラーリングは明確に分ける。
* **UI/UX:** モバイルの小さな画面でも操作しやすいよう、手札やスペルスロットのUIは画面下部に大きく配置。CD進行を視覚的にわかりやすく表示（円形ゲージ、数字カウントダウン等）。
* **サウンド:** 敵が砕け散る際の「ザクッ」「ドガァーン」といった重低音の効いた効果音を最重視。BGMはステージごとに異なるが、戦況が激化するにつれテンポが上がるインタラクティブな構成。スペル発動時の時間停止には専用のSE（時が止まる音）を用意。

## 10. 開発環境とアーキテクチャ
* **ゲームエンジン:** Godot Engine 4.x
* **開発アプローチ:**
    * 複雑な物理2Dコリジョン（当たり判定）を避け、シンプルな距離計算とレイキャストによる軽量な判定システムを自作する。これによりモバイルでも数百体のユニットの激突を処理落ちなしで実現する。

## 11. 開発マイルストーン（スモールステップ）
* **Phase 1: コアプロトタイプ（MVP）**
    * Godotでの2D基本画面構築。基本ユニット（歩兵等）の前進、攻撃、死亡処理。
    * CDベースのカード自動召喚システムの実装。手札 → CD完了 → 召喚 → 捨て札 → ドローの循環。
* **Phase 2: プレイヤー操作とスペルの実装**
    * ユニットの前後移動操作の実装。
    * スペルスロットシステム（時間停止 + 効果発動）の実装。
    * 敵ウェーブシステムの構築。
* **Phase 3: ローグライク構造の実装**
    * StS式マップ（分岐ノード）の実装。
    * ショップ、休憩、イベント、レリックシステムの構築。
    * カード獲得・除去・アップグレードの実装。
* **Phase 4: 手触りと爽快感のブラッシュアップ（Juiciness）**
    * ノックバック計算の精緻化、連鎖爆発エフェクト・ヒットストップの追加。
    * 寿命（導火線UI）システムの導入。

* **Phase 5: コンテンツ拡充と強敵の実装**
    * ボス戦とボス固有ギミックの実装。

## 12. 想定Q&A（システム・開発方針について）

### 【ゲーム体験・コアシステムに関するQA】

**Q. リアルタイム戦闘だと忙しすぎて、戦略を考える暇がないのでは？**
**A.** スペル発動時の「時間停止」がその解決策です。プレイヤーはいつでもスペルを使って時間を止め、戦場の状況を確認した上で効果を発動できます。ユニットの召喚はCDベースで自動なので操作負荷は低く、プレイヤーが能動的に行うのは「スペルの使用タイミング」と「ユニットの前後移動」のみ。リアルタイムの緊張感と、時間停止による思考の余裕を両立した設計です。

**Q. ユニットの召喚タイミングをプレイヤーが直接コントロールできないのは、ストレスになりませんか？**
**A.** むしろこれがローグライク的な面白さの核心です。CDの長さが異なるカードをどう組み合わせてデッキを構築するか、手札の巡り順を見越してプランを立てるか、が戦略の本質になります。「完全にコントロールできないからこそ、デッキ構築で確率を最適化する」というStS的な思考の楽しさが生まれます。

**Q. 当たり判定（物理衝突）をなくすことで、キャラが重なる「団子化」による見た目のチープさはどう解決しますか？**
**A.** 配置時や進行中にユニットのX軸・Y軸へ微細なランダムオフセット（揺らぎ）を加え自然なバラけ方を演出します。描画順では「小型ユニットの手前表示優先（Z-Index調整）」や、Y座標に応じた奥・手前のソートを適用し、視認性と「群れ感」を両立させます。

**Q. 妨害カード（呪い）は単なるストレスになりませんか？**
**A.** StSの呪いカードと同じ設計思想です。呪いはデッキを汚染しますが、ショップで有料除去できますし、「呪いを逆利用するレリック」も用意します。呪いの存在がショップでの除去判断、イベントでのリスクテイク判断を生み、ローグライクとしての深みに直結します。

### 【技術・パフォーマンスに関するQA】

**Q. モバイル向けとのことですが、大量のユニットと砕け散るエフェクト（ゴア表現）で処理落ちしませんか？**
**A.** ユニット同士の物理演算をカットしたことでCPU負荷は劇的に下がっています。描画面ではGodotの「GPUParticles2D」を活用し、画面外オブジェクトの即時消去や「オブジェクトプーリング」を徹底して軽量かつ派手な画面を維持します。

**Q. 画面構成が「下部40%UI、上部60%戦闘」とのことですが、縦持ちですか？横持ちですか？**
**A.** スマホの「縦持ち（ポートレート）」を想定しています。片手で操作できる手軽さを最優先します。上部60%の戦闘画面は、カメラのズームアウトや自動追従で視認性を確保します。

### 【リプレイ性・マネタイズに関するQA】

**Q. ローグライクとしての「永続的な成長（メタプログレッション）」要素はありますか？**
**A.** はい。周回プレイで得たポイントを使い、新たなカードやスペル、初期レリックをアンロックする要素を用意します。「新しい戦術やシナジーの幅が広がる」アンロック方式にし、プレイヤースキルとデッキ構築の楽しさを損なわないよう設計します。

**Q. 収益化（マネタイズ）のモデルはどう考えていますか？**
**A.** 買い切り型（プレミアムモデル）を想定しており、価格は500円前後を予定しています。ゲーム内広告やPay to Win要素は一切排除。DLCとして追加コンテンツ（新種族ユニット群、新ボス、追加ステージ等）を販売し長く遊べるタイトルに育てます。

### 【プロジェクト管理・開発体制に関するQA】

**Q. なぜ開発エンジンにGodotを採用したのですか？**
**A.** チーム構成（完全初心者3人）と開発期間を考慮したベストな選択です。Godotは軽量で2DゲームのUI・スプライト制御に極めて優れています。GDScriptは直感的で学習コストが低く、AI（生成AIアシスタント）との親和性も高いため、プロトタイプの高速イテレーションに最適です。

**Q. 完全初心者3人で、かつAIを活用して開発する上で、最も警戒しているリスクは何ですか？**
**A.** 「AIが生成したブラックボックスのコード蓄積」です。機能ごとに細かく分けてコードを書かせ、出力されたコードのロジックを3人でレビューしてから組み込むルールを徹底します。GitHubによるバージョン管理も行います。

**Q. ドット絵においてAI生成・自作・購入アセットを混在させると世界観が崩れませんか？**
**A.** 開発初期に厳格なアートガイドライン（カラーパレット、主線ルール、光源方向、ゴア表現トーン）を策定します。最終的に手作業レタッチやGodot上のポストプロセッシングを通すことで統合します。

## 13. プロからの質問・疑問・提案

### 【コアシステムに関する質問・提案】

**Q1. ユニット召喚時の挙動：「自動前進」vs「待機（その場で停止）」のどちらを基本にしますか？**
話し合いの議題に挙がっていましたが、結論が未記載です。これはゲームの根幹に関わる超重要な決定です。

* **自動前進（推奨案）:** 召喚後、敵方向へ自動的に歩き始める。射程内に敵が入ったら立ち止まって攻撃。EpicWar3と同じ挙動で直感的。プレイヤーは必要に応じて「後退」指示で引き戻す。
* **待機:** 召喚されたらその場に留まる。プレイヤーが「前進」指示を出すまで動かない。よりコントロール性が高いが操作負荷が上がる。
* **提案:** **基本は「自動前進」をデフォルトにし、ユニットごとの特性として「待機型（砲台や壁役）」を一部に設定する**のがベストだと考えます。これにより、前線に送り出すアタッカーは放っておいても機能し、壁や砲台は配置戦術としてプレイヤーの意図が反映されます。

**Q2. CDの「開始タイミング」はいつですか？**
手札に来た瞬間からCDがスタートするのか、それとも前のカードが召喚された後からスタートするのか。これは戦略の根幹を左右します。

* **手札到着時からスタート（推奨）:** 手札に入った瞬間からCDが回り始める。複数のカードが同時にCD完了する可能性があり、"ラッシュ感"が生まれる。
* **逐次スタート:** 1枚ずつ順番にCDが回る。テンポは安定するが、大量召喚の爽快感は減る。
* **提案:** 手札到着時から全てのカードのCDが**同時に回り始める方式**が良いと考えます。「CD2秒の歩兵」「CD5秒の弓兵」「CD10秒の重騎兵」が手札にあれば、2秒→5秒→10秒と順にユニットが展開されていく心地よいテンポが生まれます。これはEpicWar3の「安い兵→高い兵の順に出す」感覚と一致します。

**Q3. 手札の枚数上限はありますか？**
CD完了でカードが消費されてすぐドローされるため、手札枚数は原則一定に保たれるはずですが、何枚を想定していますか？

* **提案:** 手札は**5枚**が適切だと考えます。5枚のCDが同時進行するので、召喚ラッシュのピーク時もプレイヤーが視認・管理しやすい量です。レリックやイベントで「手札+1」「手札-1」の変動があると戦略の幅が広がります。

**Q4. 「ユニットの前後移動」の操作感は具体的にどうイメージしていますか？**
モバイルのリアルタイム操作で「個別ユニットをタップして移動」は、大量ユニットがいる画面では操作ミスが頻発するリスクがあります。

* **提案A（個別移動）:** ユニットタップ → 前進/後退ボタン。精密だが忙しい。
* **提案B（グループ移動・推奨）:** 画面上のエリアをスワイプすると、そのゾーンにいるユニットがまとめて前進/後退する。直感的かつ操作ミスが起きにくい。
* **提案C（全軍一括）:** 「全軍前進/全軍後退」ボタン1つ。シンプルだが戦術性が低い。
* **補足:** 実際にUIプロトタイプを作って操作感を試すフェーズを早期に入れることを強く推奨します。

**Q5. 「ユニットの召喚位置」はどこになりますか？**
CDが完了して自動召喚される際、ユニットは戦場のどこに出現しますか？

* **案A:** 常に自陣（画面左端）に出現し、そこから前進。
* **案B:** プレイヤーが事前に召喚ポイントを設定できる。
* **提案:** **常に自陣からの出現（案A）**がシンプルで、"自動前進で前線に向かう"という流れと噛み合います。「どこに出すか」の操作を排除したことが今回の設計の核心なので、召喚位置の指定を復活させるとコンセプトがブレます。

### 【デッキ構築・ローグライク構造に関する質問・提案】

**Q6. 初期デッキの構成はどうしますか？**
旧仕様では「見習い兵15枚」でしたが、マージシステムが撤廃された今、初期デッキをどう設計するのが最適でしょうか？

* **提案:** **基本的なユニットカード10枚（歩兵3、盾兵3、弓兵2、回復兵1、自爆兵1）**くらいのバランスから始め、早期のカード獲得とショップで自分のビルドの方向性を見出していく流れが良いです。StSも初期デッキは素朴ながら各クラスの味を感じさせる構成ですし、同様のアプローチが有効です。

**Q7. デッキの最低/最大枚数の制限はどうしますか？**
旧仕様では「15枚下限」でしたが、マージ撤廃に伴い再設計が必要です。

* **提案:** **最低枚数を削除し、デッキは「最低5枚」まで圧縮可能**にするのはいかがでしょうか。StSでもデッキ圧縮は重要戦略の一つです。ただし枚数が少なすぎると山札の循環が速すぎて呪いカードの影響が極端に大きくなるため、自然と「10〜15枚」あたりが最適になるようバランス設計できます。あるいは上限を**25〜30枚**程度に設けてデッキの無限肥大を防ぐのも一案です。

**Q8. カードのアップグレード（強化）はどういう仕組みにしますか？**
休憩所で「カードのアップグレード」が選択肢に含まれていますが、旧マージシステムの代わりとして、具体的にどのような強化が入りますか？

* **提案:** StS式に、各カードに「通常版」と「アップグレード版」の2段階を設ける。アップグレードすると、CDが短縮される/攻撃力が上がる/死亡時効果が強化される、などカードごとに固有の強化が入る。視覚的にもカードに金色の枠がつくなど差別化する。

**Q9. カード獲得時の「報酬の枚数と選択肢」はどう設計しますか？**
旧仕様では「3枚から1枚を選ぶ × 3回（リロール1回可能）」でしたが、マージが無くなった今、このフローも再設計が必要です。

* **提案:** **StS標準に合わせ「3枚から1枚（またはスキップ）× 1回」**がシンプルでテンポが良いです。エリート戦で報酬を豪華にする（レリック + カード報酬）、ボス戦で強力なカードが出やすくするなど、難易度比例の報酬設計にすれば入手の喜びが増します。

### 【スペルシステムに関する質問・提案】

**Q10. スペルスロットは何枠ですか？また、スペルの入手・強化方法は？**
スペルの枠数やラインナップの増やし方が戦略性に大きく影響します。

* **提案:** **初期2枠、最大3〜4枠（レリックやイベントで拡張）**が良いバランスだと思います。スペル自体はショップやイベントで入手。スペルもアップグレード可能にし、休憩所でカードの代わりにスペルを強化できる選択肢があると、ビルドの方向性が多様になります。

**Q11. 時間停止中に「ユニットの前後移動」も可能にしますか？**
スペル発動時の時間停止中に行える操作の範囲が、ゲーム体験を大きく左右します。

* **提案A:** スペル効果の指定だけ可能。ユニット移動は不可。→ シンプルだが時間停止の価値がスペル効果のみに限定される。
* **提案B（推奨）:** スペル効果の指定 + ユニットの移動指示も可能。→ 「戦況が危ない時にスペルで時間を止め、陣形を立て直してから再開」という"立て直しのチャンス"として機能し、プレイヤーの介入感が大きくなる。ただしスペルのCDが長ければ乱用は不可。

**Q12. 時間停止の「持続時間」は無限ですか？**
プレイヤーが好きなだけ時間停止できると、難易度が極端に下がります。

* **提案A:** 無制限。プレイヤーが「再開」ボタンを押すまで停止。思考型プレイヤー向け。
* **提案B:** 時間停止は「3〜5秒間」のリアルタイム制限あり。緊張感が増す。
* **提案C（推奨）:** **無制限だが、時間停止中に行えるアクション数に上限**を設ける（例：移動指示2回 + スペル効果発動1回）。思考時間は確保しつつ、操作量には制限がかかるバランス。

### 【バランス・ゲームデザインに関する質問・提案】

**Q13. 「マナ」の概念は完全に撤廃されましたか？**
旧仕様では「4マナ/ターン」でしたが、CDベースのシステムに変更した今、マナやコストの概念はどうなりますか？

* **提案:** **マナは撤廃し、CDの長さがコストの代替として機能させる**のが最もシンプルで美しい設計です。強力なユニット = CDが長い。弱いユニット = CDが短い。これだけで「リソース管理」のゲーム性が成立します。マナを残すと管理対象が増えてモバイル向けとしての手軽さが損なわれます。

**Q14. ユニット同士の射程・攻撃対象の優先順位はどうなりますか？**
近接・遠距離の概念があるなら、攻撃対象の選択ロジックが重要です。

* **提案:** **最前列（最も手前にいる敵）を優先攻撃**がデフォルト。一部のユニットには「後衛狙い（最もHPが低い敵を優先）」「範囲攻撃（密集地点を優先）」などの特殊効果を持たせてバリエーションを出す。

**Q15. レーンは本当に「1本」でいいですか？**
1レーンだと、戦術が「前後の押し引き」だけに限定されるリスクがあります。

* **提案:** **初期プロトタイプは1レーンで問題ありません。**ただし、将来的な拡張として「上下2〜3レーン」を追加することで、「どのレーンに戦力を集中させるか」という戦略の幅が爆発的に広がります。ボス戦の固有ギミックとして「上下レーンの分岐」を用意するのも面白いでしょう。ただしモバイルの画面サイズを考えると複数レーンは視認性が厳しくなるため、慎重に検討が必要です。

**Q16. 敵ユニットの「予兆（インテント）」は残しますか？**
旧仕様ではターン制の時止めフェーズで「次の20秒の敵出現予告」がありましたが、リアルタイムになった今はどう置き換えますか？

* **提案:** **画面右端に「次のウェーブまでのタイマー」と「次ウェーブの敵アイコン（シルエット）」を常時表示する**のが良いです。これにより「次にどんな敵が来るか」を把握した上でスペルのCD管理やユニット移動の判断が行え、ただのリアクション型ゲームではなく「先読み」の戦略性が維持されます。

**Q17. ランの長さ（1プレイの所要時間）はどのくらいを想定していますか？**
StSは1ラン30〜60分ですが、モバイルだと長すぎるという意見もあります。

* **提案:** **1ラン20〜30分**を目標にするのが良いと考えます。通勤や休憩時間に1ランが完結するのが理想。マップのノード数やボスまでの階層数で調整可能です。中断・再開機能は必須。

**Q18. 難易度調整の仕組みはどう考えていますか？**
StSにはアセンション（難易度レベル）がありますが、本作にも同様の仕組みを入れますか？

* **提案:** **クリアするごとに1段階ずつ上がるアセンション方式**をそのまま採用するのが最もリプレイ性を高めます。高アセンションでは「呪いカードの混入率上昇」「敵のHP・ATK増加」「ショップ価格上昇」「初期HP減少」など段階的にハンデが増える設計。

### 【UI/UX・モバイル操作に関する質問・提案】

**Q19. CDの視覚化はどうしますか？**
手札5枚のCDが同時に進行する → それが一目でわかるUIが非常に重要です。

* **提案:** 各カードに**円形のラジアルゲージ（時計の針のようにCDが減っていく）**を重ねて表示し、残りCD秒数を数字でも併記する。CD完了時には「光る・震える」ようなマイクロアニメーションで"召喚準備完了"を知らせる。カードが「ザッ」とフレームアウトしてユニットが戦場に現れるトランジションアニメーションも入れると爽快感が増します。

**Q20. 戦場（上部60%）のカメラ制御はどうしますか？**
横スクロールレーンなので戦場全体が一画面に収まらない可能性があります。

* **提案:** **基本は「戦線（最前列同士がぶつかっているポイント）」に自動追従**し、プレイヤーがスワイプで左右にスクロールもできるハイブリッド方式。ピンチでズームイン/ズームアウト対応。スペルの時間停止中はフリースクロールにして全体を確認できるようにするのが良いです。

### 【マネタイズ・ビジネスに関する提案】

**Q21. TGS応募を目指す上で、「体験版ビルド」の最低要件は何だと考えていますか？**
1年以内にTGSインディー部門に出展するなら、体験版として「何が遊べる状態」を目指すべきかの定義が重要です。

* **提案:** **以下を「TGS体験版」の最低ラインとして設定**するのがリアルです：
    * 最低1キャラクター（ユニット種族テーマ）が使える
    * 3〜5ステージ + ボス1体
    * StS式マップの序盤1フロア分
    * カード15〜20種 + スペル3〜5種 + レリック5〜10種
    * CDベースの戦闘 + スペル時間停止 + ユニット移動が安定動作
    * 1ラン10〜15分で完結する体験版専用ルート

**Q22. 仮に将来的にマルチプレイ要素を入れるとしたら、どんな形式が考えられますか？**
現時点では予定になくとも、設計段階で考慮しておくと後の拡張が楽になります。

* **提案:** **非同期対戦（ゴーストバトル）**が最も現実的です。他プレイヤーのデッキ構成 + AIが操作するゴースト軍と対戦する形式。リアルタイム対人はCDベースの本作の仕組みと噛み合いにくいため推奨しません。あるいは**スコアアタック型ランキング**（同じシード値のランで獲得スコアを競う）もStS系との相性が良いです。

### 【総合的な懸念点と提案】

**Q23. 「EpicWar3の気持ちよさ」と「StSの戦略性」のバランスは、本当に両立できますか？**
これは本作最大のデザインチャレンジです。正直に述べると、EpicWar3は「とにかく大量に出して蹂躙する」快感が本質で、StSは「少ないリソースで最適解を探る」快感が本質。両者は相反する要素を含んでいます。

* **懸念:** ユニットの召喚がCDで自動化された結果、「大量にユニットが湧き出る無双の快感」が薄まる可能性があります。CDが長いとテンポが悪く、短いと無双だが戦略性が消える。
* **提案:** **「波」のデザイン**で両立させます。普段はCDに従って1体ずつ召喚されるが、「ラッシュイベント」や「特定のレリック効果」で「手札のCDを全て0にする」ような爆発的なタイミングを用意する。その瞬間だけ5体同時に召喚され、EpicWar3的な無双の快感が味わえる。これを上手く散りばめると「緩急のある面白さ」が生まれます。

**Q24. ユニットの「前後移動」は、本当にゲームを面白くしていますか？**
プレイヤー操作として「スペルの時間停止」と「ユニットの前後移動」が挙げられていますが、後者がゲーム体験にどれだけ寄与するかは検証が必要です。

* **懸念:** リアルタイムで数十体のユニットの位置を操作するのは、モバイルでは非常に煩雑。かといって全軍一括だと戦術性が低い。
* **提案:** **Phase 1では「前後移動なし（完全自動戦闘）」で試作し**、スペルの時間停止だけでゲームが成立するか検証してください。面白さが足りなければ「全軍一括移動」→「グループ移動」と段階的に操作を追加する。初期から前後移動を前提にシステムを組むと、不要だった場合に大きな手戻りが発生します。

## アーカイブ

### 2026/03/07 - 大規模仕様変更
話し合いの結果、以下の大幅な変更をゲームデザインに反映：
* **マージ（進化）システムの撤廃：** 同カード2枚で自動マージ→Lv2、の仕組みを完全に除外。
* **「時止め配置」フェーズの撤廃：** ターン制の「時間停止→配置→時間進行20秒」のサイクルを廃止。
* **CDベースのリアルタイム戦闘に移行：** 各カードに固有のCDがあり、CD完了で自動召喚される方式に変更。
* **プレイヤー操作の再定義：** 「スペルの使用（時間停止）」と「ユニットの前後移動」の2つに限定。
* **スペルの別枠管理化：** スペルはデッキに混ぜず、固定スロット + CD方式で管理。
* **戦闘外はStS仕様を全面採用：** マップ移動、イベント、ショップ、休憩、レリック等のStS式ローグライク構造を導入。
* **妨害カード（呪い）の導入：** デッキに混入してプレイヤーを妨害するカード（旧「見習い兵ジレンマ」の代替）。
