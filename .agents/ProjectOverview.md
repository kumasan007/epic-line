# ゲームコンセプト
**『横スクロールハスクラ防衛ゲーム × 時間停止ユニット配置 × デッキ構築ローグライク』**

## 1. ゲームの概要
本作は、大軍勢がぶつかり合い砕け散る「横スクロール防衛ゲーム」の圧倒的なハクスラ感に、「ローグライク・デッキ構築」と「同種ユニットの自動マージ（進化）」を融合させた新感覚タクティクスゲームです。
『Epic War 3』『Slay the Spire』『Teamfight Tactics』、そして無双系アクションの爽快感を抽出し、モバイル向けに最適化して構築します。

* **画面構成:** 画面下部40%を「手札・マナ等のUI領域」、上部60%を横1レーン「戦闘画面（横分割グリッド）」として使用。
* **世界観とグラフィック:** 中世ダークファンタジーのピクセルアート。大量のユニットが死ぬ際に派手に砕け散る（ゴア・破片表現）視覚的なカタルシスを追求。
* **ハクスラの視覚化と軽量化:** モバイル最適化のため、ユニット同士の「当たり判定（物理的な渋滞）」は排除。代わりに、重い一撃による「敵の吹き飛ばし」と、密集した敵が「連鎖的に砕け散る表現」で圧倒的な爽快感を生み出す。

## 2. コア・ゲームループ（2フェーズ制）
戦闘は以下の「時止め」と「時間進行」を繰り返します。敵拠点のHPをゼロにすれば勝利、自拠点が破壊されれば敗北となります。

### ① 【時止め：ドロー＆配置フェーズ（カウンター思考）】
* **リソースとドロー:** 毎ターン「4マナ」が回復し、手札を「5枚」引く。使いきれなかったマナは次ターンへ持ち越せない。
* **山札と墓地の循環（StS仕様）:** 配置したカードも、使用せず手札に残ったカードも、ターン終了時にすべて「墓地」へ。山札が0になった際、墓地をシャッフルして新たな山札とする。
* **敵の予兆（インテント）:** 「次の20秒で、どの敵が、どの位置に召喚されるか」が事前表示される。
* **陣形構築:** マナを消費し自軍エリアにユニットや魔法を配置。ユニットは射程に入った瞬間に立ち止まり攻撃を開始する。

### ② 【時間進行：20秒間の大乱戦フェーズ（観戦と祈り）】
* **フェーズ移行時の処理:** 時間が進行したら、残っている手札を全て墓地に送り、新たに山札からカードを引く。マナも回復するが、進行中はカードの配置や使用は不可。
* **自動戦闘の観戦:** 時間が動き出し、両軍が一斉に激突する。プレイヤーは基本操作を行わず、自分の組んだ陣形が敵を蹂躙する様を**「観戦」**する。
* **倍速とリセット:** 最大2倍程度の倍速機能あり。20秒経過するとすべての動きが止まり、再びフェーズ①へ戻る。

## 3. 本作のキラーシステム（戦術を深める要素）
* **A. 「死」のリソース化:** 死亡時周囲ダメージ（自爆）や死亡時マナ回復により、意図的に生贄にして重いカードをプレイするコンボが可能。
* **B. 「寿命」と退場スキル:** 寿命（稼働時間）を迎えた瞬間に強力なスキルを放つ。寿命持ちには導火線UIが付き、「あと少し生き残ってくれ！」という祈りを視覚的に煽る。
* **C. 団子化対策とAoE（範囲攻撃）:** 当たり判定がないため、足の速い遠距離ユニットなどは一箇所に重なる（団子化）。敵側はこれに対する強烈なカウンターとして「密集地点を焼き払うAoE攻撃」を多用してくるため、配置場所やユニット構成を散らす戦術が求められる。

## 4. デッキ構築と部隊編成システム（Out-of-Battle）
本作は**「ゴールド（通貨）」の概念が存在しません**。純粋なデッキの取捨選択のみで部隊を洗練させていきます。

* **カード獲得（3×3ドラフト）:** ステージクリア後、提示される3枚のカードから1枚を選ぶプロセスを「3回」繰り返す。各回につき1度だけリロール（引き直し）が可能。
* **2枚自動マージによる進化:** デッキ内に同じカードが「2枚」揃った瞬間、自動的に「Lv2のカード1枚」に進化する。Lv2が2枚（計4枚）揃えば最大レベルの「Lv3」となる。
* **初期デッキと完全自由な「破棄（削除）」:**
    * 初期デッキは、マージ（進化）しない最弱カード**「見習い歩兵×5、見習い盾兵×5、見習い弓兵×5（全コスト1）」の計15枚**でスタート。
    * 非戦闘時であれば、不要なカードは**いつでも・何枚でも無料で「破棄（デッキから削除）」**できる。
* **最低枚数（15枚）と「見習い兵」のジレンマ:**
    * デッキの総枚数は「15枚」を下回ってはならない。カードの破棄や自動マージによって14枚以下になった場合、不足分だけ自動的に前述の「見習い兵」が補充される。
    * **戦術的ペナルティ:** 見習い兵は極めて弱いため、手札に来るとドロー枠を1つ無駄にする（使わなくても墓地に行き、再び山札に戻ってデッキを汚染し続ける）。そのため「無計画にカードを破棄して精鋭だけにすると、逆に大量の見習い兵が混ざってデッキが機能不全に陥る」という防衛ゲーム特有のジレンマが発生する。

## 5. ユニットおよび魔法のステータス定義
ユニットと魔法（設置型スペル等）は、以下の共通ステータスフォーマットで管理されます（魔法の場合は移動速度0など、用途に応じて数値を調整）。

| ステータス名 | 英語表記 | 説明 |
| :--- | :--- | :--- |
| **コスト** | Mana Cost | 召喚・発動に必要なマナ消費量。 |
| **体力** | HP / Health Points | 0になると砕け散る（死亡）。魔法の場合は「攻撃対象にならない設定（無敵）」または「破壊可能なオブジェクト」として機能する。 |
| **攻撃力** | ATK / Attack Power | 敵に与える基本ダメージ量。 |
| **攻撃範囲** | AoE | ダメージを与える範囲。 |
| **防御力** | DEF / Defense | 敵から受けるダメージを軽減する割合、または固定値。 |
| **移動速度** | SPD / Movement Speed | レーンを進む速さ。魔法や防衛施設の場合は0。 |
| **寿命** | Lifespan / Duration | 盤面に存在できる秒数（秒）。寿命が尽きるとHPに関わらず退場する。魔法の持続時間にも適用される。 |
| **スポーン時効果** | On-Spawn / Battlecry | 配置され、時間進行がスタートした瞬間に発動する固有能力。 |
| **寿命時効果** | On-Expire | 寿命（秒数）を迎えて退場する際に発動する能力。魔法の最低保証（マナ回復等）や、時限爆弾の起爆などに使われる。 |
| **死亡時効果** | On-Death / Deathrattle| HPが0になって敵に破壊された際に発動する能力（自爆ダメージ、デバフ散布など）。 |
| **ノックバック率** | Knockback Chance | 攻撃ヒット時、敵を吹き飛ばす確率（%）。 |
| **ノックバック力** | Knockback Power | 敵をどれだけ後方へ吹き飛ばすかの強さ。本作のハクスラ感の要。 |
| **ノックバック方向**| Knockback Direction| 吹き飛ばすベクトル（真後ろ、斜め上など）。この角度により、後続の敵を巻き込んで砕け散る連鎖表現が変わる。 |
| **スタン率** | Stun Chance | 攻撃ヒット時、敵の行動を完全に停止させる確率（%）。 |
| **スタン効果時間** | Stun Duration | スタン状態が持続する時間（秒）。 |
| **ノックバック耐性**| KB Resistance | 敵からの吹き飛ばしを無効化、または距離を軽減する割合（%）。重装兵のライン維持に必須。 |
| **スタン耐性** | Stun Resistance | スタン状態を無効化、または時間を軽減する割合（%）。 |
| **特殊効果** | Special / Passive | 常時発動する固有能力（例：通常攻撃がAoEになる、貫通攻撃、特定種族特効など）。 |

## 6. 強敵・ボス戦の仕様
* **ボスウェーブ:** 20秒で倒しきれなかった巨大ボスは、その場に留まり次のターンの配置フェーズへ持ち越される。
* **戦術的スタンピード（HP、経過時間トリガー）:**
    * **例:** 敵拠点やボスのHPが閾値（50%など）を下回った瞬間、予兆になかった大量の敵が急激に出現する。HPバーの「警告線（ドクロマーク）」により、プレイヤーは閾値を超えるタイミングを意図的にコントロール（コントロールド・パニック）できる。

## 7. ターゲット層とプレイ体験の目標（UX目標）
* **ターゲット層:** 『Slay the Spire』のような思考のパズル的楽しさと、『Epic War 3』や無双系のような「画面を埋め尽くす敵をなぎ倒す」視覚的快感の両方を味わいたい、ミッドコア層以上のゲーマー向け。
* **至高の快感（脳汁ポイント）:** 
    * 敵の大群が「渋滞」している箇所へ、絶妙なタイミングで「ノックバック特大」の重攻撃ユニットやAoE魔法を配置し、敵陣がドミノ倒しのように連鎖的に砕け散る瞬間。
    * 「あと1秒で自拠点が落ちる！」という絶望的な状況で、狙いすましたデッキ構築のコンボ（死亡時効果の連鎖など）が決まり戦況が完全逆転するカタルシス。

## 8. アートスタイルとサウンドディレクション
* **ビジュアル:** ダークファンタジーテイストのピクセルアート。ユニットは小さめ（Epic War 3リスペクト）だが、攻撃エフェクトや死亡時のディスメンバーメント（四肢飛散、破片飛散）はあえて過剰に描き、爽快感を強調する。スマホ画面での視認性を高めるため、自軍・敵軍のシルエットやカラーリングは明確に分ける。
* **UI/UX:** モバイルの小さな画面でも操作しやすいよう、手札やマナのUIは画面下部に大きく配置。ドラッグ＆ドロップによる直感的な配置操作。
* **サウンド:** 敵が砕け散る際の「ザクッ」「ドガァーン」といった重低音の効いた効果音を最重視。BGMは「時止めフェーズ」では静かで重厚な思考用BGM、「時間進行フェーズ」では一気にテンポが上がるインタラクティブな構成。

## 9. さらなる拡張アイデア（レリック / シナジー要素）
将来的な拡張性として以下の要素を追加し、ローグライクとしてのリプレイ性を高める。
* **遺物（アーティファクト）システム:** ボス討伐時や特定イベントで獲得できるパッシブアイテム。「全近接ユニットのノックバック率+10%」「ターン開始時、マナ上限+1だが手札-1」など、ゲームごとの戦術（ラン）を決定づける要素。

## 10. 開発環境とアーキテクチャ
* **ゲームエンジン:** Godot Engine 4.x
* **開発アプローチ:** 
    * 複雑な物理2Dコリジョン（当たり判定）を避け、シンプルな距離計算とレイキャストによる軽量な判定システムを自作する。これによりモバイルでも数百体のユニットの激突を処理落ちなしで実現する。

## 11. 開発マイルストーン（スモールステップ）
* **Phase 1: コアプロトタイプ（MVP）**
    * Godotでの2D基本画面構築。基本ユニット（歩兵等）の前進、攻撃、死亡処理。
    * 「時止め」と「時間進行（20秒）」のフェーズ切り替えループの実装。
* **Phase 2: デッキ構築とローグライク要素の実装**
    * 手札ドロー、マナ消費によるユニット配置、ターン終了時の墓地送りの実装。
    * リザルト画面でのカード3択ピック、同カード2枚での自動マージ（レベルアップ）処理。
* **Phase 3: 手触りと爽快感のブラッシュアップ（Juiciness）**
    * ノックバック計算の精緻化、連鎖爆発エフェクト・ヒットストップの追加。
    * 寿命（導火線UI）システムの導入。
* **Phase 4: コンテンツ拡充と強敵の実装**
    * 敵の予兆（インテント）UIの可視化。
    * ボスウェーブと「戦術的スタンピード」の実装。UI/UXの最終研磨。
